# Encryption

> Encrypt storage values and optionally keys using `encryptedStorage`.

::note
This feature is available in unstorage v2 (alpha) and Nitro v3.
::

## Overview

`encryptedStorage` wraps any storage instance with end-to-end encryption. You can compose it on top of an existing storage and all read/write operations are transparently encrypted and decrypted.

- **Values** are encrypted using **XChaCha20-Poly1305** with random nonces
- **Keys** can optionally be encrypted using **AES-256-GCM-SIV** with deterministic nonces

Encryption is powered by [`@noble/ciphers`](https://github.com/paulmillr/noble-ciphers) which is a zero-dependency library that works across Node.js, Bun, Deno, etc.

## Usage

`encryptedStorage(storage, options)`{lang=ts}

```ts
import { createStorage, encryptedStorage } from "unstorage";

const storage = encryptedStorage(createStorage(), {
  encryptionKey: "<base64-encoded-256-bit-key>",
});

await storage.setItem("secret", "my sensitive data");
await storage.getItem("secret"); // "my sensitive data"
```

### Generating an encryption key

The encryption key must be a **base64-encoded 256-bit (32 bytes) key**.

::tabs
  ::tab{label="OpenSSL"}
  ```bash
  openssl rand -base64 32
  ```
  ::
  ::tab{label="Node.js"}
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
  ```
  ::
  ::tab{label="JavaScript"}
  ```ts
  const key = btoa(String.fromCodePoint(...crypto.getRandomValues(new Uint8Array(32))));
  ```
  ::
::

::warning
Store the encryption key securely (e.g. environment variable). If the key is lost, encrypted data cannot be recovered.
::

## Encrypting keys

By default, only values are encrypted — storage keys remain in plaintext. If key names themselves contain sensitive information (e.g. user IDs, email addresses), enable `encryptKeys` to encrypt them as well:

```ts
const storage = encryptedStorage(createStorage(), {
  encryptionKey: "<base64-encoded-256-bit-key>",
  encryptKeys: true,
});

await storage.setItem("users/alice@example.com", { role: "admin" });

// Keys are transparently encrypted/decrypted
await storage.getItem("users/alice@example.com"); // { role: "admin" }
await storage.getKeys(); // ["users:alice@example.com"]
```

In the underlying storage, the key is stored as an opaque `$enc:` prefixed base64 string instead of the plaintext path.

::tip
Key encryption uses **AES-256-GCM-SIV** with a deterministic nonce so that the same plaintext key always produces the same ciphertext — this is what makes lookups possible. GCM-SIV is nonce-misuse-resistant, making this safe.
::

::warning
When `encryptKeys` is enabled, `getKeys()` fetches and decrypts **all** keys from the underlying storage before filtering. This may impact performance with large key sets.
::

## Options

### `encryptionKey`

- **Type:** `string`
- **Required:** `true`

Base64-encoded 256-bit encryption key used for both value and key encryption.

### `encryptKeys`

- **Type:** `boolean`
- **Default:** `false`

When enabled, storage keys are also encrypted using AES-256-GCM-SIV. Encrypted keys are prefixed with `$enc:` and stored as URL-safe base64.

### `sivNonce`

- **Type:** `string`
- **Default:** `"ThtnxLK9eCF4OLMg"`

Custom base64-encoded nonce used for AES-GCM-SIV key encryption. Must be 16 bytes. Override this to use a unique deterministic nonce per application.

### `encryptedKeyPrefix`

- **Type:** `string`
- **Default:** `"$enc:"`

Custom prefix for encrypted keys. Useful when you need to distinguish encrypted keys from different applications sharing the same storage backend.

## Combining with `prefixStorage`

`encryptedStorage` composes with other storage utilities like `prefixStorage`:

```ts
import { createStorage, encryptedStorage, prefixStorage } from "unstorage";

const storage = createStorage();
const prefixed = prefixStorage(storage, "app");
const encrypted = encryptedStorage(prefixed, {
  encryptionKey: "<base64-encoded-256-bit-key>",
});

await encrypted.setItem("x", "hello");
await encrypted.getItem("x"); // "hello"
```

## Raw values

`encryptedStorage` supports `setItemRaw` and `getItemRaw` for encrypting binary data:

```ts
const raw = new TextEncoder().encode("binary-data");
await storage.setItemRaw("bin", raw);
const result = await storage.getItemRaw("bin"); // Uint8Array
```

## How it works

### Value encryption

Values are encrypted with **XChaCha20-Poly1305** using a random 24-byte nonce generated via `crypto.getRandomValues()`. The encrypted value and nonce are stored together as a `StorageValueEnvelope`:

```ts
interface StorageValueEnvelope {
  nonce: string;
  encryptedValue: string;
}
```

### Key encryption

When `encryptKeys` is enabled, keys are encrypted with **AES-256-GCM-SIV** using a fixed nonce. GCM-SIV is nonce-misuse-resistant, making it safe to use with a deterministic nonce — which is required so that the same plaintext key always maps to the same encrypted key for storage lookups.

Encrypted keys are prefixed with `$enc:` and encoded as URL-safe base64.
